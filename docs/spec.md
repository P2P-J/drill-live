# Driller Live — 완전 개발 명세서
> 다른 AI 또는 개발자가 이 문서만 읽고 전체 시스템을 구현할 수 있도록 작성된 기술 명세서

---

## 0. 프로젝트 한 줄 정의

유튜브 라이브 방송 화면 자체가 게임이다. 시청자의 슈퍼챗/채팅이 실시간으로 게임에 영향을 주는 **브라우저 기반 무한 채굴 자동화 라이브 게임**이다.

---

## 1. 전체 시스템 아키텍처

```
[YouTube Live]
  슈퍼챗 / 채팅 발생
        │
        ▼
[Streamer.bot] ← 로컬 실행, 무료
  - YouTube Super Chat 트리거 감지
  - WebSocket 서버 내장 (ws://localhost:8080)
        │
        ▼
[Node.js 이벤트 서버] ← 로컬 실행
  - youtube-chat npm으로 채팅 폴링
  - 트리거 파싱 (커맨드 인식)
  - 쿨다운 관리
  - 이벤트 큐 처리 (동시 다발 후원)
  - !boss 커맨드 자동 응답 봇
        │ WebSocket (ws://localhost:3001)
        ▼
[Phaser.js 게임] ← localhost:5173 (Vite dev server)
  - 무한 채굴 자동 진행
  - 바이옴 색상 블렌딩
  - 광물 생성 / 채굴 애니메이션
  - 폭발 이펙트
  - 보스 이벤트 연출
  - HUD 표시
        │
        ▼
[OBS Studio]
  - 브라우저 소스: localhost:5173
  - 게임 화면을 유튜브로 송출
        │
        ▼
[YouTube Live 방송]
```

## 4. 게임 핵심 메커니즘

### 4-1. 기본 게임플레이

게임은 **완전 자동 진행**이다. 스트리머가 직접 조작하지 않는다.
드릴이 화면 중앙에서 아래 방향으로 자동으로 땅을 파내려간다.
시청자는 후원과 채팅으로만 게임에 개입한다.

**기본 루프:**
```
자동 채굴 진행
→ 광물 획득 시 카운터 증가
→ 누적 골드로 업그레이드 (스트리머 수동 구매)
→ 더 빠른/넓은 채굴
→ 깊이 증가 → 바이옴 변화
→ 보스 구간 도달 → 이벤트 발생
→ 무한 반복
```

---

### 4-2. 드릴 애니메이션 — 핵심 구현 사항

**레퍼런스 분석 (ZRQYT 채널 영상 기준):**

ZRQYT는 마인크래프트 봇이 곡괭이로 블록을 채굴하는 방식이다.
우리는 이것을 Phaser.js로 재현한다.

**곡괭이/드릴 구조:**
- 손잡이(handle): 갈색/금색 대각선 막대
- 날(bit): 업그레이드 단계에 따라 색상이 다른 L자형 날
  - 1단계(목재): 갈색
  - 2단계(돌): 회색
  - 3단계(철): 은색
  - 4단계(금): 노란색
  - 5단계(다이아): 청록색

**채굴 애니메이션 사이클 (반드시 이 순서로 구현):**

```
① 접근 (Approach)
   드릴 날 끝이 타일 표면으로 이동
   duration: 약 150ms
   easing: Sine.easeIn

② 접촉 (Contact)
   드릴 날 끝이 정확히 타일 표면에 닿음
   이 순간에만 채굴 판정 발생
   타일에 크랙 스프라이트 오버레이 (0→1→2→3→4단계)

③ 반동 (Rebound)
   블록에 닿은 후 약간 뒤로 튕겨나옴
   duration: 약 80ms
   easing: Bounce.easeOut
   이동 거리: 약 10~15px

④ 반복
   블록이 완전히 파괴될 때까지 ①~③ 반복
   드릴 파워에 따라 반복 횟수 결정
   - 1단계: 5회 타격으로 파괴
   - 5단계: 1회 타격으로 파괴

⑤ 파괴 (Break)
   타일 파괴 시 픽셀 파편 파티클 방사
   광물 타일이면 해당 광물 카운터 +1
   드릴이 다음 타일로 이동
```

**중요: 드릴은 빈 공간(검은 배경)을 그냥 통과한다. 타일이 있는 곳에만 채굴 판정이 발생한다.**

**드릴 이동 방향:**
기본적으로 아래 방향으로 이동하며, 아래가 막히면 좌우를 탐색한다.
자동 채굴이므로 최단 경로로 아래를 향해 이동한다.

**드릴 회전:**
드릴이 이동 방향에 따라 rotation이 바뀐다.
- 아래로 이동: 0도
- 왼쪽 대각선: -45도
- 오른쪽 대각선: +45도
회전 전환은 Tween으로 부드럽게 처리한다.

### 4-3. 타일 시스템

**타일 크기:** 32×32px (Phaser 기본)

**타일 종류:**
```
- 일반 타일 (돌, 흙 등): 드릴로 채굴 가능
- 광물 타일: 채굴 시 해당 광물 카운터 증가
- 바위 타일 (rock): 드릴로 불가, 폭탄 트리거로만 파괴
- 빈 공간 (void): 검은 배경, 드릴 그냥 통과
```

**타일 파괴 표현 (크랙 단계):**
```
타일 위에 크랙 스프라이트를 오버레이로 겹친다
단계 0: 크랙 없음 (원본 타일)
단계 1: 약한 크랙 (25% 데미지)
단계 2: 중간 크랙 (50% 데미지)
단계 3: 심한 크랙 (75% 데미지)
단계 4: 완전 파괴 → 타일 제거 + 파편 파티클
```

**무한 타일맵 구현:**
- 화면에 보이는 구간만 타일을 생성한다 (뷰포트 기반 청크 방식)
- 위로 스크롤된 타일은 제거하여 메모리 절약
- 청크 크기: 화면 높이 × 2 분량을 미리 생성
- 드릴이 화면 하단 70% 지점에 도달하면 화면을 스크롤하고 새 청크 생성

### 4-6. 업그레이드 시스템

채굴로 쌓인 골드로 업그레이드. **스트리머가 수동 구매.**
자동 구매 없음.

| 장비 | 효과 | 최대 단계 | 드릴 날 색 변화 |
|------|------|-----------|---------------|
| 드릴 파워 (Drill Power) | 타격 횟수 감소 (채굴 속도 UP) | 5단계 | 갈색→회→은→금→청록 |
| 드릴 범위 (Drill Range) | 한 번에 파는 타일 수 증가 | 3단계 | |
| 연료 탱크 (Fuel Tank) | 연료 용량 증가 (방치 시간 증가) | 5단계 | |
| 인벤토리 (Cargo) | 광물 보관 칸 수 증가 | 5단계 | |
| 엔진 (Engine) | 이동 속도 증가 | 3단계 | |

### 4-7. 보스 이벤트 시스템

**보스는 경량 이벤트 연출 방식이다. 복잡한 전투 시스템 없음.**
보스 등장 → 후원으로 체력 깎기 → 결과 처리의 단순 구조.

**보스 등장 위치:**

| 보스 이름 | 등장 깊이 | 바이옴 |
|----------|-----------|--------|
| Mega Mole | ~9,000km | Earth |
| Crystal Golem | ~45,000km | Crystal Cave |
| Abyss Kraken | ~90,000km | Abyssal Sea |
| Ancient Treant | ~450,000km | Ancient Forest |
| Magma Dragon | ~950,000km | Magma Core (최강) |

**보스 예고 시스템 (BossTracker):**

```
보스까지 1,000km 남음:
→ HUD에 주황색 "BOSS INCOMING" 텍스트 표시
→ 봇 자동 채팅: "⚠️ Boss in ~1,000km!"

보스까지 500km 남음:
→ HUD 깜빡임 시작 + 경고음
→ 봇 자동 채팅: "⚠️ Boss in ~500km! Get ready!"

보스까지 100km 남음:
→ 화면 테두리 붉게 점멸
→ 봇 자동 채팅: "🔴 BOSS ALMOST HERE!"

보스 등장:
→ 화면 흔들림 (Phaser camera shake)
→ 배경 붉은 빛으로 플래시
→ 보스 체력바 화면 상단에 등장
→ 드릴 자동 정지 또는 속도 감소
→ 봇 자동 채팅: "🔥 BOSS APPEARED! Donate to fight!"
```

**!boss 채팅 커맨드:**
시청자가 언제든 `!boss`를 입력하면 봇이 자동 응답:
```
"🦷 Next boss: [보스 이름] — [남은 km]km away!
 Current depth: [현재 깊이]km | Boss at: ~[등장 깊이]km"
```

**보스 전투 흐름:**

```
1. 보스 등장 연출 (3초)
2. 체력바 표시 (최대치 = 100)
3. 제한 시간 카운트다운 (TBD, config로 조정 가능)
4. 후원/채팅 트리거로만 체력 감소:
   - BOSS BOMB ($2): 체력 -10
   - BOSS NUKE ($10): 체력 -40

성공 (체력 0):
→ 보스 처치 폭발 이펙트
→ 희귀 광물 클러스터 대량 스폰
→ 후원자 이름 목록 5초 화면 표시
→ 드릴 재개

실패 (시간 초과):
→ 드릴 속도 -50% 페널티, 30초
→ 리셋 없음, 채굴 계속 진행
```

## 5. 트리거 시스템

### 5-1. 설계 원칙

```
채팅 트리거:
- 전체 채널 기준 쿨다운 (한 명이 쓰면 모두 대기)
- 효과 지속 중 동일 커맨드 재입력 → 무시 (중첩 없음)
- 효과 종료 후에만 재작동
- 게임에 큰 영향 없는 시각/속도 연출 위주

슈퍼챗 트리거:
- 쿨다운 없음, 즉시 발동
- 같은 효과 중복 시 지속시간 갱신 (중첩 아님)
- NUKE 계열은 Queue에 쌓아 순차 발동

구독/멤버십 트리거:
- 즉시 발동, 쿨다운 없음
- 후원자 이름 화면 표시 병행
```

### 5-2. 채팅 트리거

[광물 아이콘] [수량]

Depth: 7,760km
Biome: 🌍 Earth
Gold: 12,450
```

```
⚠ BOSS IN ~1,240km
Next: Mega Mole

(보스 1,000km 이내 진입 시 표시 시작)

## 12. 중요 설계 원칙 (개발 시 반드시 준수)

1. **게임 화면에 명령어 목록 넣지 않는다.** 설명란/고정댓글에 넣는다.
2. **후원 발생 시 화면에 이름+이펙트만 표시한다.** UI는 최소화.
3. **모든 수치(깊이, 보스 등장 km)는 config 파일로 분리한다.** 하드코딩 금지.
4. **게임은 절대 리셋되지 않는다.** 보스 실패도 페널티만 있고 계속 진행.
5. **언제 들어와도 3초 안에 이해 가능한 구조를 유지한다.**
6. **OBS 브라우저 소스 기준 해상도: 1920×1080, 60fps.**
7. **서버는 로컬에서만 동작한다.** 클라우드 배포 불필요.
8. **채팅 트리거는 게임에 큰 영향을 주지 않는다.** 시각적 몰입 위주.
9. **슈퍼챗 트리거는 즉시 발동한다.** 쿨다운 없음.
10. **동시 다발 후원은 큐(Queue)로 순차 처리한다.** 렌더링 부하 방지.
