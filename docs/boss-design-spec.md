# 🐉 보스 이미지 디자인 스펙

여기 적힌 규격대로 PNG 파일을 만들어서 폴더에 넣어두면 게임이 자동으로 사용합니다.
파일이 없는 보스는 자동으로 procedural 기본 디자인이 그려져요 (개발용 placeholder).

## 📁 파일 위치

```
public/assets/bosses/
├── megaMole.png
├── crystalGolem.png
├── abyssKraken.png
├── ancientTreant.png
└── magmaDragon.png
```

> 파일을 폴더에 드롭한 뒤 페이지 새로고침하면 즉시 적용.

## 🎨 공통 규칙

| 항목 | 값 | 비고 |
|---|---|---|
| 포맷 | **PNG** | 투명 배경 필수 (RGBA) |
| 색상 모드 | 8-bit RGBA | |
| 배경 | **완전 투명** | 흰 배경 ❌ |
| 발 위치 | **이미지 맨 아래** | 보스의 발/베이스(뿌리·촉수·꼬리)가 이미지 하단에 닿게 |
| 정렬 | 가로 중앙 | 보스의 중심축이 이미지 가로 중앙 |
| 자세 | 정면 or 3/4 측면 | 좌우 이동 시 자동 flip되니 한쪽 방향만 그리면 됨 |
| 여백 | 좌우 최소 | 이미지 양옆 투명 여백 최소화 (그래야 충돌 정확) |

## 📐 권장 크기

각 보스의 비율/크기 가이드. 너무 크면 아레나에 꽉 차고, 너무 작으면 안 보임.

| 보스 | 권장 W × H | 비율 | 컨셉 |
|---|---|---|---|
| **megaMole** | 500 × 400 | 가로형 | 갈색 두더지 — 큰 발톱 옆으로 펼침 |
| **crystalGolem** | 400 × 600 | 세로형 | 보라 결정 거인 — 키 큰 휴머노이드 |
| **abyssKraken** | 600 × 550 | 정사각형에 가까움 | 8개 촉수 펼친 심해 괴물 |
| **ancientTreant** | 450 × 650 | 세로형 | 고대 나무 — 가지 팔 + 뿌리 다리 |
| **magmaDragon** | 650 × 500 | 가로형 | 날개 펼친 용 — 가로 비행 자세 가능 |

아레나 크기는 약 **704 × 768 px** (11타일 × 12타일). 보스가 이 안에 충분히 들어가야 함.

## ✅ 체크리스트 (파일 보내기 전에)

- [ ] PNG 포맷? (JPG/WebP 안 됨)
- [ ] 배경 투명? (흰색 배경 X)
- [ ] 발/베이스가 이미지 맨 아래?
- [ ] 가로 중앙 정렬?
- [ ] 권장 크기 비율 비슷?
- [ ] 파일명 정확? (대소문자 구별 — camelCase)

## 🔄 게임 내 동작

- 보스가 등장하면 **아레나 바닥에 발 닿아 서 있음**
- 좌우로 움직이며 **이동 방향에 따라 sprite flip** (좌→우 갈 때 → 그림 좌우 반전)
- 드릴이 보스에 부딫치면 자동 데미지
- HP 0 → 회전하며 폭발 사라짐
- 시간 초과 → 위로 도망

## 🎨 디자인 톤 가이드 (참고)

게임이 9:16 세로 화면에서 OBS로 캡처되므로:

- **실루엣이 강하게 보이도록** — 윤곽선 두껍게 (3~5px)
- **하이라이트/그림자 명확** — 평면적이지 않게
- **눈이 큼지막하게** — 작은 화면에서도 표정 보이게
- **컬러 팔레트는 바이옴에 맞춰**:
  - Earth (mole) → 갈색·황토색 베이스
  - Crystal Cave (golem) → 보라·청록 발광
  - Abyss (kraken) → 짙은 청록·심해 발광
  - Forest (treant) → 짙은 녹·갈색 + 녹색 발광 룬
  - Magma (dragon) → 적·주황·검정 + 노란 불꽃
